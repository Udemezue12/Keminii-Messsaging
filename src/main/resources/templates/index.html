Built for messages that can't wait — or leak

Four properties, held at once, on every connection.

Encrypted

Every frame is encrypted before it leaves the client. Kemini never stores or logs plaintext message bodies.

Fast

A persistent WebSocket connection skips the handshake overhead of request-response APIs, so messages arrive as they happen.

Reliable

Dropped connections reconnect automatically, and undelivered frames are queued so a flaky network doesn't cost you a message.

Secured

Origin checks, message signing, and per-connection rate limits run underneath the socket, without adding a step to your code.

Free and public

No signup, no API key, no rate card. Kemini is open for testing and building — the only thing to configure is where you connect from.

Connect from localhost

Kemini's public test socket is reachable on your own machine at port 8989. Three steps, one socket.

01

Open the socket

Point a WebSocket client at ws://localhost:8989. No headers or credentials required.

02

Send a frame

Write a message to the socket. Kemini encrypts it in transit and echoes a delivery receipt.

03

Listen for messages

Incoming frames are decrypted and handed to your onmessage handler as plain text.

connect.js 
const socket = new WebSocket("ws://localhost:8989");

socket.onopen = () => {
  socket.send(JSON.stringify({
    type: "message",
    body: "hello, kemini"
  }));
};

socket.onmessage = (event) => {
  console.log("received:", event.data);
};
